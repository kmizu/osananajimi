# 第10話　部活の仲間たち

　金曜日の放課後、プログラミング部では新歓コンパが開かれていた。美月の入部を祝う小さなパーティーだ。僕のThinkPadの画面には、今週の成果物が表示されている。美月が最初に書いたHello Worldプログラムから、昨日完成させた花のパターン描画まで。この一週間で、彼女の成長は目覚ましかった。

「桜井さん、一週間お疲れ様でした」

　山田部長がお疲れ様の挨拶をしてくれた。部長の手元には、美月の入部届と、彼女が作成したプログラムのポートフォリオがある。

「ありがとうございます。とても楽しい一週間でした。プログラミングって、こんなに創造的なものだとは思いませんでした」

　美月は嬉しそうに答えた。テーブルの上には彼女のノートが開かれていて、Pythonの基本構文がきれいにまとめられている。変数の宣言、条件分岐、ループ処理。一週間でここまで理解するなんて、本当に優秀だ。

　部室には簡単なお菓子とジュースが用意されていて、みんなでくつろいでいる。壁には、来月のIOI国内予選に向けたスケジュール表と、アルゴリズム理論の参考図書リストが貼られている。

「美月の上達が早くて驚いてるよ。もうfor文とwhile文を使い分けて、効率的な反復処理を書けてる」

　田村先輩が言った。先輩の手元には、美月が昨日書いたフィボナッチ数列のプログラムがある。再帰処理と動的プログラミングの違いを理解して、メモ化による最適化まで実装していた。

「本当に。もう基本的なプログラムは書けるようになったし、アルゴリズムの時間計算量の概念も理解してる。O(n)とO(log n)の違いを説明できるなんて、初心者とは思えない」

　慎也も賛成した。慎也は競技プログラミングが得意で、特にグラフ理論のエキスパートだ。彼が美月のことを認めるということは、本当にセンスがあるということだ。

「拓海先輩が優秀な先生だからです。アルゴリズムを日常生活に例えて説明してくださるので、とても分かりやすくて」

　美月が僕の方を見て言った。その瞬間、胸がドキドキする。美月の真剣な眼差しが、僕だけに向けられている。心拍数が上がるのを感じながら、平静を装う。

「いや、美月が頑張ってるからだよ。質問も的確だし、一度教えたことをすぐに応用できる。ソートアルゴリズムなんて、僕が説明したバブルソートを聞いただけで、クイックソートの改良点まで自分で考えついたじゃないか」

　僕は少し照れながら答えた。実際、美月の理解力には毎回驚かされる。プログラミングの論理構造を、まるで詩の韻律を理解するように自然に習得していく。

「お互いを褒め合って、いいコンビだね」

　山田先輩が笑いながら言った。

「そういえば、美月は将来どんなことがやりたいの？ITエンジニアとか、研究者とか考えてる？」

　田村先輩が美月に聞いた。先輩は大学でコンピュータサイエンスを専攻する予定で、将来の進路についてよく相談に乗ってくれる。

「まだよく分からないんですが、プログラミングを学んでみて、色々な可能性があることが分かりました。技術を使って、人と人とを繋げることができるんですね」

「例えば？具体的にはどんなアプリケーションを作ってみたい？」

「ウェブサイトを作って、文学作品をもっと多くの人に伝えたり、インタラクティブな詩の朗読サイトを作ったり。あとは、おばあちゃんでも使いやすいスマートフォンアプリとか」

　美月は目を輝かせて話した。プログラミングの技術的な面だけでなく、それを使って社会をどう良くするかまで考えている。この視点は、競技プログラミングに没頭している僕たちには新鮮だった。

「いいね。プログラミングは手段だから、それを使って何をしたいかが大切なんだ。僕たちはどうしても技術的な課題ばかりに目が向きがちだけど、美月の視点は大事だね。ユーザビリティとかアクセシビリティとか、そういう観点が抜けがちになる」

　山田先輩が言った。部長は将来、フロントエンドエンジニアになりたいと言っている。ユーザーインターフェースの設計や、ユーザーエクスペリエンスの向上に興味がある。

「私、文芸部でも活動してるので、詩や小説をもっと多くの人に読んでもらえるようなウェブサイトが作れたらいいなって思います。響きのあるフォントで表示したり、朗読音声を組み合わせたり、読者がコメントを書き込めるような機能があったら」

「素晴らしいアイデアだね」

　田村先輩が感心した。

「拓海、美月のその夢、手伝ってあげたら？君はバックエンドの処理が得意だし、美月はユーザーインターフェースの感覚がある。いいコンビになりそうだ」

　慎也が僕に向かって言った。慎也の提案に、僕の心が躍る。美月と一緒にプロジェクトを進められるなんて、考えただけで胸が高鳴る。

「もちろん。美月が作りたいものがあれば、いつでも協力するよ。RESTful APIの設計とか、データベース構築とか、僕が担当できる部分はたくさんある」

「ありがとうございます」

　美月は嬉しそうに微笑んだ。

「ところで、来月の文化祭の準備、そろそろ始めないといけないな。去年はタワーディフェンスゲームを展示したけど、今年はもっとインタラクティブなものにしたい」

　山田先輩が話題を変えた。壁に貼られた昨年の文化祭の写真を指差す。来場者がプログラムをいじって遊んでいる様子が写っている。

「今年は何をやるんですか？AI関係とか、機械学習のデモとかでしょうか？」

　美月が興味深そうに聞いた。美月は文芸部だけど、技術トレンドにも敏感だ。最近のAIブームについてもよく質問してくる。

「まだ決まってないんだ。何かアイデアある？プログラミング初心者の視点で提案してもらえると助かる」

「そうですね」

　美月は少し考えてから答えた。

「来場者の人たちに、プログラミングの楽しさを体験してもらえるような展示はどうでしょうか？ビジュアルプログラミングツールを使って、ブロックを組み合わせるだけで簡単なアニメーションが作れるような」

「いいアイデアだね。Scratchみたいなツールを使えば、プログラミング未経験の人でも楽しめる」

「具体的にはどんなコンテンツを？デモ用のサンプルプロジェクトも必要だよね」

　田村先輩が聞いた。先輩はWebデザインが得意で、ユーザーインターフェースの設計によく口を出してくる。

「簡単なゲームを作って、その場でパラメータを変更してもらうとか。キャラクターの移動速度を変えたり、背景色を変えたり。プログラムのコードを直接見せて、『ここの数字を変えると、こんな風に動きが変わります』って説明できたら」

「面白そう。リアルタイム協調編集とかできたら、複数人で同時にプログラムをいじれるね。Google Docsみたいに」

　慎也が乗り気になった。慎也はクラウド技術に詳しくて、最新のWeb技術についてよく教えてくれる。

「それなら、来場者の人たちも楽しめるし、プログラミングに興味を持ってもらえるかも。美月のアイデアは、技術的に高度すぎず、でも魅力的だ。プロジェクト管理の観点から見ても実現可能性が高い」

　僕も賛成した。実際、このプロジェクトはアジャイル開発の良い題材になりそうだ。スクラム手法を使って、短いイテレーションで機能を追加していけばいい。

「じゃあ、それで進めてみようか」

　山田先輩が決めてくれた。

「美月、企画の中心になってもらえる？プロダクトオーナーとして、要件定義とユーザーストーリーの作成を担当してもらいたい」

「私がですか？まだ入部したばかりなのに。プロダクトオーナーって、責任重大ですよね」

「だからこそ、初心者の視点が大切なんだ。僕たちは技術者の視点に偏りがちだから、ユーザー目線でのフィードバックが必要。アジャイル開発では、ステークホルダーとの継続的な対話が重要だから」

　田村先輩が説明してくれた。先輩は将来、プロジェクトマネージャーになりたいと言っている。

「分かりました。頑張ります。でも、技術的な実装については、皆さんにお任せすることになりますが」

　美月は決意を込めて答えた。その真剣な表情を見ていると、僕の胸が熱くなる。美月がチームの中心になって、みんなで協力して一つのものを作り上げる。そんな未来が楽しみでならない。

「みんなで協力して、素晴らしい展示を作ろう」

　山田先輩の言葉で、みんなの気持ちが一つになった。

　コンパが終わって、片付けをしていると、美月が僕のところに来た。テーブルの上のお菓子の袋をまとめながら、小声で相談してくる。

「拓海先輩、本当に大丈夫でしょうか？私が企画の中心だなんて。プロジェクト管理とか、スケジュール調整とか、経験がないんです」

「大丈夫だよ。美月のアイデアは素晴らしかったし、ユーザビリティの感覚が鋭い。技術仕様書は僕たちが書くから、美月には要件定義書の作成をお願いしたい」

「でも、技術的なことは全然分からないし。APIとかデータベースとか、まだ勉強中です」

「それは僕たちがサポートするから。美月には、どうやったら来場者の人たちに楽しんでもらえるか考えてもらいたいんだ。ユーザーインターフェースの設計とか、ユーザーエクスペリエンスの向上とか。それが一番重要な部分だから」

　僕がそう言うと、美月は少し安心したような顔をした。でも同時に、責任の重さも感じているようだった。その表情を見ていると、美月を支えてあげたいという気持ちが強くなる。

「ありがとうございます。頑張ります」

「一緒に頑張ろう」

　部室を出て、僕たちは一緒に帰路についた。

「今日のコンパ、楽しかったです」

「美月が加わって、部の雰囲気も明るくなったよ」

「そうでしょうか？」

「うん。みんな美月のことを気に入ってるし」

　美月は嬉しそうに微笑んだ。

「文化祭の企画、頑張らないといけませんね」

「みんなで協力すれば、きっといいものができるよ」

「はい。楽しみです」

　夕日が校舎を照らしていて、とても美しかった。京都の古い街並みと、学校の現代的な建物のコントラストが、オレンジ色の光に映えている。

「美月」

「はい？」

「プログラミング部に入ってくれて、ありがとう。君がいてくれると、僕たちの視野が広がる。技術だけじゃなくて、それを使う人のことを考えられるようになった」

「こちらこそ、ありがとうございます。毎日がとても充実してます。プログラミングを通して、論理的思考力も身につきましたし、チームで協力することの楽しさも学べました」

　美月がそう言ってくれて、僕は心から嬉しかった。美月の成長を間近で見られることが、僕にとっても大きな喜びになっている。教えることで、僕自身の理解も深まっているし、美月の質問は常に本質を突いている。

　最近、美月と過ごす時間がどんどん増えている。部活も一緒、帰り道も一緒、勉強も一緒。プログラミングの課題を一緒に解いたり、アルゴリズムについて議論したり。文化祭のプロジェクト計画も、二人で相談しながら進めることになる。

　これが当たり前になっているけれど、僕にとってはとても特別なことだ。美月と一緒にコードを書いている時間は、まるで時間が止まったような感覚になる。隣に座って、同じ画面を見つめながら、一緒に問題を解決していく。その共同作業が、僕にはとても心地よい。

　美月の家の前まで来ると、僕たちは立ち止まった。

「今週もお疲れ様でした」

「美月も、お疲れ様」

「来週からは文化祭の準備も始まりますね」

「そうだね。忙しくなりそうだ」

「でも、楽しそうです」

　美月は目を輝かせて言った。

「きっと楽しいよ。美月と一緒なら、絶対にいいプロジェクトができる。君のユーザビリティの感覚と、僕の技術力を組み合わせれば」

　僕がそう言うと、美月は少し頬を赤らめた。夕日のせいかもしれないが、その表情がとても可愛く見える。

「私も、拓海先輩と一緒なら頑張れます。ペアプログラミングとか、コードレビューとか、まだ慣れませんが、一緒に学んでいけたら」

「じゃあ、また月曜日に」

「はい。また月曜日」

　美月が家に入っていくのを見送ってから、僕は自分の家に向かった。

　今週は美月がプログラミング部に入部して、とても充実した一週間だった。美月が部の仲間になって、僕も嬉しい。彼女がいることで、チーム全体の雰囲気も明るくなった。技術的な議論だけでなく、ユーザーエクスペリエンスについて話し合う機会も増えた。

　来月の文化祭に向けて、みんなで協力して頑張ろう。美月の企画を成功させるために、僕も全力でサポートしたい。スクラム手法を使って、効率的にプロジェクトを進められるはずだ。美月をプロダクトオーナーとして、僕はスクラムマスターの役割を担おう。

　部屋に戻って、僕は文化祭の企画についてアイデアを考え始めた。ThinkPadを開いて、プロジェクト管理ツールのTrelloにボードを作成する。「文化祭展示プロジェクト」というタイトルで、To Do、Doing、Doneの列を作る。美月の夢を叶えるお手伝いができるかもしれない。

　最初のタスクカードを作成する。「要件定義書作成 - 担当：美月」「技術仕様書作成 - 担当：拓海」「UI/UXデザイン - 担当：田村先輩」。アジャイル開発の手法を使って、スプリント計画を立ててみよう。

　窓の外を見ると、美月の家の明かりが見えた。きっと美月も、文化祭のことを考えているのだろう。プロダクトオーナーとしての責任を感じて、ユーザーストーリーの作成に取り組んでいるかもしれない。

　これから、美月とたくさんの思い出を作っていけそうだった。プログラミングを通じて、一緒に何かを創り上げる喜びを共有できる。それが、僕にとって最高の幸せだった。
